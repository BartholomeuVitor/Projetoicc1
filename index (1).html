Git e GitHub

Git

Git é um sistema de controle de versão de arquivos; em outras palavras, é responsável por guardar o histórico de alterações sempre que alguém modificar algum arquivo que está sendo monitorado por ele.

GitHub

GitHub é uma plataforma de hospedagem de código-fonte com controle de versão usando o Git. Ele permite colaboração entre desenvolvedoresutilitários ou qualquer usuário cadastrado na plataforma contribuam em projetos privados e/ou Open Source de qualquer lugar do mundo.

Comandos Git

Inst
Comando Exibição
init Esta tag é usada para iniciar um repositório Git em um diretório. Quando você executa git init, ele cria um novo repositório Git na pasta especificada..
clone Usado para criar uma cópia de um repositório Git existente. Quando você clona um repositório, você está criando uma cópia local de todo o histórico de commits, branches e arquivos do repositório remoto.
branch Uma branch em Git é uma linha independente de desenvolvimento. Você pode criar novas branches para trabalhar em novas funcionalidades sem interferir no código principa.
checkout Essa tag é usada para mudar de branch ou restaurar arquivos para um determinado commit. Você pode usá-la para navegar entre branches existentes ou criar uma nova branch a partir de um commit específico.
status Ao executar git status, o Git mostra o estado atual do seu repositório. Ele lista os arquivos modificados, os arquivos prontos para serem adicionados ao próximo commit e outras informações úteis.
diff Esta tag é usada para mostrar as diferenças entre arquivos. Quando você executa git diff, o Git exibe as alterações feitas em arquivos que ainda não foram adicionados ao próximo commit.
add Usado para adicionar alterações ao próximo commit. Você usa git add para incluir as modificações feitas em arquivos específicos no próximo commit.
commit Esta tag é usada para confirmar as alterações feitas no repositório. Quando você executa git commit, você está criando um novo commit contendo as modificações adicionadas com git add.
push Utilizado para enviar commits locais para um repositório remoto. Você usa git push para enviar as alterações locais para um servidor Git remoto, como o GitHub ou o GitLab.
pull Essa tag é usada para obter as alterações do repositório remoto e mesclá-las com o seu repositório local. O comando git pull é uma combinação de git fetch e git merge.
revert Usado para desfazer um commit específico, criando um novo commit que reverte as alterações introduzidas pelo commit especificado.
merge Utilizado para combinar duas ou mais branches. O comando git merge integra as alterações de uma branch em outra.
stash Usado para armazenar temporariamente alterações que ainda não estão prontas para serem commitadas. É útil quando você precisa mudar de branch ou resolver um conflito antes de fazer commit das alterações.
rm Esta tag é usada para remover arquivos do repositório Git. Quando você executa git rm, você está removendo um arquivo do controle de versão do Git.
config Utilizado para configurar as opções do Git. Você pode usar git config para definir configurações como nome de usuário, endereço de e-mail, editor padrão, etc.
reset Essa tag é usada para desfazer alterações no repositório. Você pode usar git reset para redefinir o HEAD para um commit específico, desfazendo commits ou alterando o estado do repositório.
remote Usado para gerenciar repositórios remotos. Com git remote, você pode listar, adicionar e remover repositórios remotos.
fetch Essa tag é usada para buscar alterações de um repositório remoto para o repositório local, sem mesclá-las com o seu trabalho atual. As alterações são armazenadas no seu repositório local, mas não são aplicadas no seu código atual.
show Utilizado para mostrar informações sobre commits, tags ou branches específicos. Com git show, você pode ver as alterações introduzidas por um commit, o conteúdo de uma tag, entre outras informações.
help Essa tag é usada para obter ajuda sobre comandos Git específicos. Você pode executar git help para obter informações detalhadas sobre como usar um comando Git específico.
rebase Usado para modificar o histórico de commits. Com git rebase, você pode reorganizar, combinar e editar commits antes de enviá-los para um repositório remoto.
blame Esta tag é usada para mostrar quem modificou cada linha de um arquivo e em qual commit a modificação foi feita. É útil para rastrear a autoria de mudanças específicas.
tag Utilizado para criar, listar e deletar tags no repositório Git. As tags são usadas para marcar pontos específicos na história do seu projeto, como versões lançadas.

Acesso GitHub

Para configurar e acessar o GitHub, siga os passos abaixo:

  • Passo 1: Crie uma conta no GitHub.
  • Passo 2: Instale o Git em seu computador.
  • Passo 3: Configure seu nome de usuário e email: 
    git config --global user.name "Seu Nome"
git config --global user.email "seuemail@example.com"
  • Passo 4: Crie um novo repositório no GitHub e clone-o em seu computador: 
    git clone https://github.com/seu_usuario/seu_repositorio.git
